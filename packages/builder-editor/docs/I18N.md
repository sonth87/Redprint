# i18n Support in Builder Editor

## Overview

The Builder Editor supports internationalization (i18n) with both **nested** and **flat** translation formats.

## Built-in Languages

- **English** (`en`) - Default
- **Vietnamese** (`vi`)

## Adding Custom Languages

### Method 1: Nested Format (Recommended)

```json
// translations/ko.json
{
  "common": {
    "save": "저장",
    "cancel": "취소"
  },
  "toolbar": {
    "undo": "실행 취소",
    "redo": "다시 실행"
  }
}
```

```tsx
import koTranslations from "./translations/ko.json";

<BuilderEditor
  locale="ko"
  i18nResources={{
    ko: { translation: koTranslations }
  }}
/>
```

### Method 2: Flat Format

```json
// translations/ko-flat.json
{
  "common.save": "저장",
  "common.cancel": "취소",
  "toolbar.undo": "실행 취소",
  "toolbar.redo": "다시 실행"
}
```

```tsx
import koFlatTranslations from "./translations/ko-flat.json";

<BuilderEditor
  locale="ko"
  i18nResources={{
    ko: { translation: koFlatTranslations }
  }}
  // keySeparator defaults to '.' which supports both formats
/>
```

### Method 3: Multiple Languages

```tsx
import jaTranslations from "./translations/ja.json";
import zhTranslations from "./translations/zh.json";
import frTranslations from "./translations/fr.json";

<BuilderEditor
  locale="ja"
  i18nResources={{
    ja: { translation: jaTranslations },
    zh: { translation: zhTranslations },
    fr: { translation: frTranslations },
  }}
/>
```

## Props

### `locale?: string`
The active language code (e.g., `"en"`, `"vi"`, `"ko"`, `"ja"`).

### `i18nResources?: Record<string, { translation: Record<string, unknown> }>`
Additional translation resources to merge with built-in `en` and `vi`.

### `i18nKeySeparator?: string | false`
- **Default: `'.'`** - Supports both flat (`"toolbar.undo"`) and nested (`{ toolbar: { undo: "..." } }`) formats
- **`false`** - Disables key nesting, only flat keys work

## Example: Language Switcher

```tsx
import { i18n } from "@ui-builder/builder-editor";

function LanguageSwitcher() {
  const [locale, setLocale] = useState("en");

  const changeLanguage = (lang: string) => {
    i18n.changeLanguage(lang);
    setLocale(lang);
  };

  return (
    <BuilderEditor
      locale={locale}
      i18nResources={{
        ko: { translation: koTranslations },
        ja: { translation: jaTranslations },
      }}
    />
  );
}
```

## Notes

- Both nested and flat formats can be mixed in the same resource object
- The default `keySeparator: '.'` allows using both `t('toolbar.undo')` and nested structures
- Built-in `en` and `vi` translations are always available and serve as fallbacks
